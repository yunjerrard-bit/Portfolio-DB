## Phase Details

### Phase 1: 기반 + 단일 티커 수직 슬라이스

**Goal:** 단일 미국 티커 한 개에 대해 OHLCV 수신 → EMA/expanding-window 통계 계산 → 정적 색 베이킹 → 단일 시트 .xlsx 출력까지 vertical end-to-end 슬라이스를 완성하여 Core Value(중앙값±σ 색 신호)가 살아있음을 증명한다.
**Mode:** mvp
**Depends on:** Nothing (first phase)
**Requirements:** INPUT-01, INPUT-02, INPUT-03, INPUT-05, MKTD-01, MKTD-02, MKTD-03, COMP-01, COMP-02, COMP-03, COMP-04, COMP-05, COMP-06, TECH-01, TECH-02, TECH-03, TECH-04, TECH-05, TECH-06, SHEET-01, SHEET-02, SHEET-03, SHEET-04, SHEET-05, SHEET-06, SHEET-07, SHEET-08, COLOR-01, COLOR-02, COLOR-03, COLOR-04, COLOR-05, COLOR-06, COLOR-07, OUT-01, OUT-02, OUT-03, EXEC-01, EXEC-02
**Success Criteria** (what must be TRUE):

  1. 사용자가 `tickers.txt`에 단일 티커(예: `AAPL`) 한 줄을 적고 `uv run python main.py`를 실행하면 `output/portfolio_YYYYMMDD.xlsx`가 새로 생성된다
  2. 생성된 파일을 Excel로 열면 `AAPL` 시트가 있고 A1=티커, 3행=전체기간 누적 중앙값, 4행=전체기간 누적 표준편차, 5행=한국어 헤더, 6행부터 날짜 내림차순으로 OHLCV/EMA11·22·96·192/차이/일별 변동/일별 중앙값·표준편차가 채워져 있다
  3. 데이터 셀의 글자색·배경색이 `값 vs 그 날짜의 expanding window 중앙값±1σ/2σ` 규칙대로 정적으로 베이킹되어 있어 사용자가 셀을 클릭해 봐도 동적 조건부서식 규칙이 없다 (가장 최근/중간/가장 오래된 3개 행을 수기 검증 시 일치)
  4. 입력 파일이 비어있거나 없을 때 한국어 에러 메시지를 출력하고 0이 아닌 종료코드로 안전하게 종료한다
  5. `.env`의 `EDGAR_USER_AGENT_EMAIL`과 `OPENDART_API_KEY`가 시작 시 로드되며 비어있으면 명확한 한국어 메시지로 실패한다 (Phase 3 사전 준비)
  6. 시트에 Stochastic Slow(`%K`, `%D`)와 RSI(14) 열이 추가되어 있고 `Stoch ≤20`/`RSI ≤30`은 부드러운 초록, `Stoch ≥80`/`RSI ≥70`은 부드러운 빨강으로 정적 색 베이킹되어 있다

### Phase 2: N개 티커 스케일링 + 포트폴리오 요약 시트

**Goal:** Phase 1의 수직 슬라이스를 N개(최대 200개) 티커로 확장하고, 시트1 통합 포트폴리오 요약을 추가한다. 데이터 수신 계층을 캐시·재시도·throttle·부분데이터 검증으로 강화하여 200 티커 처리가 rate-limit 위반 없이 완료되도록 한다.
**Mode:** mvp
**Depends on:** Phase 1
**Requirements:** INPUT-04, MKTD-04, MKTD-05, MKTD-06, PORT-01, PORT-02, PORT-03, PORT-04, PORT-06, PORT-07, PORT-08, TECH-07, EXEC-03, EXEC-05
**Success Criteria** (what must be TRUE):

  1. 사용자가 `tickers.txt`에 미국·한국 혼합 N개(예: 10개 이상) 티커를 적고 `python main.py` 실행 시 모든 티커에 대해 개별 시트가 생성되고, 시트1(첫 번째 시트)에 티커명·시장(US/KR)·최신 종가·전일대비 등락률·EMA11/22/96/192 색상 신호 4셀·거래량 색상 신호·**Stochastic Slow(최신 %K)·RSI(최신값) 각 색상 신호**가 한 행씩 표시된다
  2. 시트1의 티커명 셀이 해당 종목 시트로 가는 하이퍼링크로 동작하며 시트1 상단에 실행 시각 타임스탬프가 표시된다
  3. 한 티커 수신 실패(잘못된 형식, 네트워크, 부분 데이터 <50%)는 전체 실행을 중단시키지 않고 콘솔에 한국어 경고로 기록되며 나머지 티커는 정상 처리된다
  4. 같은 날 두 번째 실행 시 OHLCV는 sqlite 디스크 캐시(24h TTL)에서 우선 조회되어 Yahoo Finance 호출이 거의 발생하지 않는다 (콘솔 로그로 cache hit/miss 확인 가능)
  5. 200 티커 처리가 max_workers=4 + 토큰버킷 throttle 하에서 Yahoo/EDGAR/DART rate-limit 위반(429/403) 없이 완료된다

### Phase 3: 기본적 분석 데이터 (EDGAR/DART/yfinance·Naver 보완)

**Goal:** 시트1에 PER/PEG/GPM/OPM 기본적 분석 지표 컬럼을 추가하고, 미국 종목은 EDGAR(edgartools)·한국 종목은 DART(OpenDartReader)를 1차 소스로, yfinance/네이버 금융을 보완 소스로 사용한다. 각 값에는 어느 소스에서 왔는지(출처)가 명시된다.
**Mode:** mvp
**Depends on:** Phase 2
**Requirements:** FUND-01, FUND-02, FUND-03, FUND-04, FUND-05, FUND-06, PORT-05
**Success Criteria** (what must be TRUE):

  1. 시트1의 각 미국 티커 행에 PER/PEG/GPM/OPM 4개 값이 채워지고 그 값이 EDGAR에서 왔음(예: `(EDGAR)` 또는 출처 컬럼)이 표시된다 (1차 소스 결손 시 yfinance로 자동 폴스루되고 `(yf)` 표시)
  2. 시트1의 각 한국 티커 행에 동등 지표가 채워지고 DART corp_code 매핑을 통해 정확한 회사 데이터가 조회되며 `(DART)` 또는 `(Naver)` 출처 표시가 보인다
  3. EDGAR 호출 헤더에 `User-Agent: ... yunjerrard@gmail.com`이 포함되고 토큰버킷이 EDGAR ≤8 req/s, DART ≤2 req/s를 강제하여 403/quota 초과 없이 완료된다
  4. 같은 주 내 두 번째 실행 시 EDGAR/DART 응답은 sqlite 캐시(7d TTL)에서 조회되어 외부 API 호출이 거의 발생하지 않는다 (분기 단위 변동을 반영한 보수적 TTL)

### Phase 4: 품질·견고성 마감

**Goal:** 일상 사용 품질로 마감. 누락·부분수신·인증실패를 콘솔 최종 요약 블록·시트1 실패행·셀 주석으로 한 곳에서 확인하고(D-01: 별도 시트 미생성), 1~5행 frozen panes를 적용하며, 콘솔 로그는 한국어로 진행률·캐시 hit/miss·실패 요약을 출력한다. 색상 톤은 강렬하지 않은 파스텔로 시각 검증한다.
**Mode:** mvp
**Depends on:** Phase 3
**Requirements:** OUT-04, EXEC-04
**Success Criteria** (what must be TRUE):

  1. 모든 시트에서 1~5행이 frozen 처리되어 사용자가 6행 이하 데이터를 스크롤해도 헤더·중앙값·표준편차가 항상 보인다
  2. 잘못된 형식 티커·부분 데이터 수신(<50%)·인증 실패·corp_code 매핑 실패 등이 (a) 콘솔 최종 실패 요약 블록 (b) 시트1 실패행 (c) 셀 주석으로 종목별 사유와 함께 확인된다 (D-01: 별도 시트 미생성)
  3. 200 티커 실행 종료 시 콘솔에 한국어 진행률(예: `[42/200] OK AAPL`), 캐시 hit/miss 통계, 최종 실패 요약 블록이 출력된다
  4. 색상 톤이 PROJECT.md 기준 "강렬하지 않은 톤"(파스텔/소프트)이며 그레이스케일 스크린샷에서도 ±1σ/±2σ 신호가 시각적으로 구분된다
