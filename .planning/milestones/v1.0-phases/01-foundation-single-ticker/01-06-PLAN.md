<objective>
사용자 수기 검증에서 발견된 출력 서식 결함 4건 (가격 소수점 2자리·쉼표·% 서식·% 소수점 2자리)을 해결한다. Phase 1 출력 품질을 사용자가 받아들일 수 있는 수준으로 마감.

배경: 01-04에서 구현된 Format 캐시(9키)는 색만 담당하고 num_format이 없어 모든 숫자가 Excel 기본 표시(가변 자리수·쉼표 없음)로 노출됨. 색 로직은 정확하므로 유지하고, num_format만 색과 직교(orthogonal)하게 추가한다.

Output: 재실행한 `output/portfolio_YYYYMMDD.xlsx`를 Excel로 열었을 때 종가 = `123.45`, 거래량 = `12,345,678`, RSI = `54.32%` 형태로 표시.
</objective>

<task id="2">
<title>sheet_per_ticker.py — column_num_format 헬퍼 + write 루프 통합</title>

<behavior>
새 헬퍼 함수 `column_num_format(col_name: str) -> str` 추가:
- col_name이 "Volume", "Volume_median", "Volume_std" → "volume"
- col_name이 "Stoch_%K", "Stoch_%D", "RSI" → "percent"
- 그 외 (Close/High/Low + EMA/DIFF/dailychg + 모든 _median/_std) → "price"

write_sheet_for_ticker 내부의 cell write 호출에서 기존 `bucket` 결정 후
`fmt_type = column_num_format(col_name)`, `fmt = formats[(bucket, fmt_type)]`로 교체.

3행(median), 4행(std) 작성 시에도 num_format 적용 — 동일한 (DEFAULT, fmt_type) 사용.
5행 헤더는 기존 "header" Format 유지 (글자 서식만, num_format 없음).
</behavior>
