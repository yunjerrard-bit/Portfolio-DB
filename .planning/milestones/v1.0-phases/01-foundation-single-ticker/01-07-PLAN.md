<objective>
사용자 결정: 모든 DIFF 차이 컬럼을 **종가 EMA 기준으로 통일**한다. 고가·저가 EMA 컬럼은 제거하고, 일변동(dailychg)도 종가 EMA 한 가지만 유지한다.

배경 (이전 동작):
- EMA: 12개 (Close/High/Low × 4 periods)
- DIFF: 12개 (각 가격 - 자기 가격의 EMA)
- dailychg: 12개 (각 가격 EMA의 .diff())

새 동작:
- EMA: **4개** (Close × 4 periods)만
- DIFF: **12개 유지** — Close/High/Low 각각이 EMA_Close_N과 비교
- dailychg: **4개** (EMA_Close_N.diff())만

총 데이터 컬럼: 124 → 76. 색 베이킹 로직과 expanding window 통계는 컬럼별 median/std 기준으로 그대로 작동 (의미는 "차이의 cumulative median ± σ"로 해석됨).
</objective>

<title>compute/ema.py — add_ema_columns 재작성 (Close EMA만, DIFF는 cross-price)</title>

<behavior>
함수 시그니처와 반환 contract:
- 입력: OHLCV DataFrame (Close, High, Low 포함)
- 출력: 입력 + 다음 신규 컬럼들의 copy:
  - EMA_Close_{11,22,96,192} (4개) — pandas .ewm(span=N, adjust=False).mean()
  - DIFF_Close_{11,22,96,192}, DIFF_High_{...}, DIFF_Low_{...} (12개) — `(df[price] - df[EMA_Close_N]) / df[EMA_Close_N]` (비율, 예: +0.0123 = +1.23%)
  - EMA_Close_{N}_dailychg (4개) — `df[EMA_Close_N].diff()`

PRICE_COLS 상수는 유지 (DIFF 용). 더 이상 High/Low의 EMA는 계산하지 않는다.

기존 함수 `compute_ema(series, span)` 유지 (단위 테스트용 export).
</behavior>

- EMA_Close_11 → "종가 EMA11"
- DIFF_Close_11 → "종가-EMA11 차이"
- DIFF_High_11 → "고가-EMA11 차이" (의미는 "고가 - 종가EMA11" / EMA_Close_11)
- DIFF_Low_11 → "저가-EMA11 차이"
- EMA_Close_11_dailychg → "종가 EMA11 일변동"

기타 _median/_std suffix 로직은 그대로 동작.

KOREAN_HEADERS 정적 dict alias도 새 layout 기반으로 재생성.

column_num_format 갱신:
- "Volume", "Volume_median", "Volume_std" → "volume"
- "Stoch_%K", "Stoch_%D", "RSI" → "percent_literal" (0~100 스케일, 리터럴 % 접미사)
- "DIFF_*", "DIFF_*_median", "DIFF_*_std" → "percent_ratio" (0~1 스케일, Excel '0.00%' 자동 ×100)
- 그 외 가격 계열 (Close/High/Low + EMA_Close_N + dailychg + 그 _median/_std) → "price"

writer.py Format 캐시 키 확장: 8 buckets × 4 fmt_types ("price","volume","percent_literal","percent_ratio") + "header" = 33키.
