<objective>
SQLite 영구 store 모듈 `fundamentals_store.py`를 신규 생성한다 — `data/fundamentals.db`(WAL)에 분기별 원천 raw long 테이블(`raw_facts`)과 델타 state 테이블(`delta_state`)을 만들고, 정정공시 덮어쓰기(upsert)·과거 분기 보존·결손 NULL 저장·last_accession CRUD·델타 hit/miss 카운터를 제공한다. 동시에 Wave 0 테스트 스캐폴드(`tests/test_fundamentals_store.py` + conftest 격리 fixture)를 만들어 FUND-07을 검증 가능하게 한다.

D-09: raw_facts의 유니크 키는 `(ticker, source, quarter, field)`이며, 정정공시(같은 분기·새 accession) 재추출 시 최신값 upsert(덮어쓰기) + accession 메타 갱신으로 처리한다(정정 이력 미보존, 매트릭스 = 최신 진실).

Purpose: 이 store는 Phase 8(지표 계산)·Phase 9(트렌드 엑셀)의 단일 원천 백엔드이며, 과거 분기가 사라지지 않는 영구 누적(TTL 없음)이 Core 가치다. 델타(Plan 03)·추출(Plan 02)·오케스트레이션(Plan 04)이 이 store 위에서 동작한다.
