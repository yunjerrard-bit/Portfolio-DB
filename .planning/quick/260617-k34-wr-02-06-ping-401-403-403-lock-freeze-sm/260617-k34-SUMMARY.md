# Phase quick-260617-k34: WR-02~06 안정성 수정 + IN-02 Summary

04-REVIEW.md 안정성 권고 5건(WR-02~06) + 정리 1건(IN-02)을 닫았다: 인증 ping 을 tenacity 재시도 + 401/403 한정 + 구조적 403 status_code 판별로 교체하고, 캐시 싱글톤 3개소를 double-checked lock 으로 보호하고, freeze/smoke 테스트에 ping·fetch_fundamentals stub 을 추가해 네트워크 0 을 보장했다.

## What Was Built

### Task A — auth_check.py (WR-02/03 + IN-02)
- `_edgar_probe`/`_dart_probe` 에 `@retry(stop_after_attempt(3), wait_exponential(multiplier=1, min=1, max=8), reraise=True)` 추가. 데코레이터 순서는 `@throttled_X` outer → `@retry` inner (market.py 패턴 동일).
- `ping_edgar`: `except httpx.HTTPStatusError as e:` 에서 `e.response.status_code` 가 401/403 일 때만 인증 실패 (403→UA 노트, 401→일반 노트). 5xx 등은 `(True, None)`. 그 외 transient 예외도 `(True, None)`.
- WR-03: `note = _EDGAR_403_NOTE if "403" in str(e) else ...` 완전 삭제 → 구조적 `status_code` 분기로 대체. 소스 `"403" in str` 무매치 확인.
- `ping_dart`: 401/403 한정 인증 실패 + 5xx/transient → `(True, None)`. status 분기(무효 키 "010" → 실패)는 유지. `except` 절 미사용 `e` 바인딩 제거(IN-02 F841).

### Task B — cache.py (WR-04)
- `_cache_lock = threading.Lock()` 추가 (`_stats_lock` 과 분리).
- `_get_cache`/`_get_fund_cache`/`_get_name_cache` 3개소를 double-checked locking 으로 교체: fast-path 무lock `if _x is None` → `with _cache_lock` → 재확인 `if _x is None` → mkdir + Cache 생성. hot path 비용 0.

### Task C — 테스트 격리 (WR-05/06)
- WR-05 (test_freeze_panes.py fixture): `ping_edgar`/`ping_dart`/`fetch_fundamentals` stub 추가.
- WR-06 (test_smoke_n_tickers.py fixture): `fetch_fundamentals` stub 추가.
- ping-failure spy(`_spy_fund`): 원본 real fetch 위임 제거 → `return None`. skip_edgar 인자 전파 단언(`c[2] is True`) 불변, 무한재귀/네트워크 0 보장.
